HTML Documents

Todo e qualquer documento envolvendo programação html deve começar com um tipo de declaração para o arquivo funcionar com as tags do HTML.

Todo documento deve começar com a declaração: <!DOCTYPE html>.

O documento começa com a tag <html> e termina com a tag </html>.

Toda a parte visível do documento em HTML estará entre as tags <body> e </body>.

Exemplo:

<!DOCTYPE html>

<html>

<body>

<p> Exemplo de paragráfo dentro das tags body. </p>

</body>

</html>

HTML Semantics

Em HTML, a semântica refere-se ao significado dos elementos e como eles se relacionam com o conteúdo e a estrutura da página.

Importância da Semântica em HTML

Usar semântica adequada em HTML é importante por várias razões:

  • Melhora a acessibilidade para pessoas com deficiências visuais, auditivas ou cognitivas.
  • Melhora a indexação e compreensão do conteúdo por parte dos motores de busca.
  • Facilita a manutenção e a colaboração no código, tornando-o mais compreensível para outros desenvolvedores.

Exemplos de Elementos Semânticos

Alguns elementos HTML que fornecem semântica clara são:

  • <header>: Define um cabeçalho ou área de introdução.
  • <nav>: Define uma seção de navegação.
  • <main>: Define o conteúdo principal da página.
  • <article>: Define um conteúdo autônomo que pode ser distribuído ou reutilizado independentemente.
  • <footer>: Define um rodapé ou área de fim de página.

Importância de um Guia de Estilo em HTML

Ter um guia de estilo em HTML traz vários benefícios, incluindo:

HTML Charsets

O charset em HTML (conjunto de caracteres) especifica o conjunto de caracteres usado na página web.

Importância do Charset em HTML

Usar o charset correto é fundamental para garantir que os caracteres sejam exibidos corretamente em diferentes navegadores e dispositivos.

Exemplos de Charsets em HTML

Alguns exemplos comuns de charsets em HTML são:

  • UTF-8: O conjunto de caracteres Unicode que suporta a maioria dos idiomas e caracteres especiais.
  • ISO-8859-1: Conjunto de caracteres ocidental que suporta a maioria dos idiomas europeus.
  • ISO-8859-2: Conjunto de caracteres central europeu.

Como Definir o Charset em HTML

O charset é especificado na tag <meta> dentro da seção <head> da página:

<meta charset="UTF-8">

HTML URL Encode

O URL encoding em HTML é o processo de converter caracteres especiais em uma URL em uma forma segura para transmissão através da internet.

Exemplos de Caracteres Especiais em URLs

Alguns exemplos comuns de caracteres especiais em URLs são:

  • Espaço: convertido para %20
  • Amperstand (&): convertido para %26
  • Acentos e caracteres não-ASCII: convertidos para sequências de escape UTF-8, como %C3%A9 para "é".

Como Fazer o URL Encode em HTML

O URL encoding pode ser feito manualmente substituindo cada caractere especial por sua sequência de escape correspondente, ou usando a função encodeURIComponent() em JavaScript.

Nesse caso iremos substituir o seguinte link:

https://www.example.com/search?q=olá mundo&pagina=1

E substituído ficaria:

https://www.example.com/search?q=ol%C3%A1%20mundo&pagina=1

Nesta URL encode, o espaço foi substituído por "%20" e o "á" foi substituído por "%C3%A1".

HTML vs XHTML

HTML (Hypertext Markup Language) e XHTML (Extensible Hypertext Markup Language) são linguagens de marcação para criar páginas da web, mas existem diferenças importantes entre elas.

Diferenças Principais

  • HTML é mais flexível e tolerante a erros, enquanto o XHTML é mais estrito e exige uma sintaxe mais rigorosa.
  • Em HTML, as tags não precisam ser fechadas e letras maiúsculas e minúsculas são ignoradas, enquanto em XHTML todas as tags devem ser fechadas corretamente e as letras maiúsculas e minúsculas são sensíveis.
  • HTML é mais compatível com navegadores antigos e pode ser interpretado de forma mais liberal, enquanto XHTML é mais adequado para ambientes XML e é analisado de forma mais rigorosa.

Compatibilidade e Uso

HTML é a versão mais comum e amplamente utilizada para criar páginas da web atualmente, devido à sua natureza mais flexível e compatibilidade com navegadores. XHTML foi uma tentativa de reformular HTML em uma sintaxe mais estrita e compatível com XML, mas não conseguiu ganhar ampla adoção.

Conclusão

A escolha entre HTML e XHTML depende das necessidades específicas do projeto e das preferências do desenvolvedor. Para a maioria dos casos, HTML é a escolha mais prática e compatível, enquanto XHTML pode ser mais adequado para ambientes XML e projetos específicos que requerem uma sintaxe mais rigorosa.

HTML Forms

Os formulários em HTML são usados para coletar informações dos usuários, como nome, senha, endereço e muito mais.

Estrutura de um Formulário HTML

Um formulário HTML é definido usando a tag <form>, dentro da qual são incluídos elementos de entrada e botões de envio:

<form action="/processar_dados" method="post">
    <label for="nome">Nome:</label>
    <input type="text" id="nome" name="nome"><br>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email"><br>
    
    <button type="submit">Enviar</button>
</form>

Principais Elementos de Formulário em HTML

  • <input>: Usado para criar campos de entrada, como texto, senha, email, checkbox, entre outros.
  • <textarea>: Usado para criar campos de texto multi-linha.
  • <select>: Usado para criar menus dropdown com opções selecionáveis.
  • <button>: Usado para criar botões de envio, reset ou botões personalizados.

Processamento de Dados do Formulário

Quando um formulário é enviado, os dados são geralmente processados por um servidor web, que pode armazenar, processar ou responder aos dados enviados pelo usuário.

Atributos de Formulário HTML

Os atributos de formulário em HTML são usados para controlar o comportamento e a funcionalidade dos formulários em uma página da web.

Principais Atributos de Formulário

  • action: Especifica para onde os dados do formulário serão enviados quando ele for submetido.
  • method: Define o método HTTP usado para enviar os dados do formulário, geralmente sendo "get" ou "post".
  • name: Define o nome do formulário, usado para identificar o formulário no código ou no lado do servidor.
  • target: Especifica o destino onde o resultado do envio do formulário será exibido, como uma nova janela ou uma frame.
  • autocomplete: Controla se o recurso de preenchimento automático de formulário deve ser ativado ou desativado.
  • enctype: Especifica o tipo de codificação de caracteres usada para enviar os dados do formulário, geralmente sendo "application/x-www-form-urlencoded" ou "multipart/form-data".
  • novalidate: Indica se o navegador deve ou não validar os campos do formulário antes do envio.

Exemplo de Uso de Atributos de Formulário

Exemplo de um formulário HTML com alguns dos atributos mencionados:

<form action="/processar_dados" method="post" name="meuForm" target="_blank" autocomplete="on" enctype="multipart/form-data" novalidate>
</form>

HTML Canvas

O elemento canvas em HTML é usado para desenhar gráficos, gráficos e outros elementos visuais em uma página da web de forma dinâmica.

Como Usar o Elemento Canvas

Para usar o elemento canvas, você precisa definir sua largura e altura, e então desenhar nele usando JavaScript:

<canvas id="meuCanvas" width="400" height="200"></canvas>

Exemplo de Desenho em Canvas

Aqui está um exemplo simples de desenho de uma linha em um canvas usando JavaScript:

var canvas = document.getElementById('meuCanvas'); var context = canvas.getContext('2d'); context.beginPath(); context.moveTo(50, 50); context.lineTo(200, 200); context.stroke();

Recursos do Canvas

O elemento canvas oferece uma ampla gama de recursos para desenho, incluindo:

  • Desenho de formas básicas, como retângulos, círculos e linhas.
  • Desenho de texto.
  • Aplicação de estilos, como cores, preenchimentos e gradientes.
  • Transformações, como rotação, escala e translação.
  • Manipulação de pixels para criar efeitos visuais.

O elemento SVG (Scalable Vector Graphics) em HTML é usado para criar gráficos vetoriais escaláveis e interativos em uma página da web.

Como Usar o Elemento SVG

Para usar o elemento SVG, você pode incluí-lo diretamente em seu código HTML:

<svg width="400" height="200"> <circle cx="100" cy="100" r="50" fill="red" /> </svg>

Exemplo de Desenho com SVG

Aqui está um exemplo simples de desenho de um círculo em SVG:

<svg width="200" height="200">
    <circle cx="100" cy="100" r="80" fill="blue" />
</svg>

Recursos do SVG

O SVG oferece uma ampla gama de recursos para criação de gráficos vetoriais, incluindo:

  • Desenho de formas básicas, como retângulos, círculos, elipses, linhas e polígonos.
  • Desenho de texto.
  • Aplicação de estilos, como cores, preenchimentos, gradientes e sombras.
  • Transformações, como rotação, escala e translação.
  • Definição de elementos interativos, como links e eventos de mouse.

HTML Media/Video

Os elementos de mídia em HTML são usados para incorporar e exibir vários tipos de conteúdo de mídia em uma página da web.

Principais Elementos de Mídia em HTML

  • <img>: Usado para exibir imagens em uma página da web.
  • <audio>: Usado para reproduzir áudio em uma página da web.
  • <video>: Usado para reproduzir vídeos em uma página da web.

Exemplo de Uso de Elementos de Mídia

Exemplo de incorporação de uma imagem, um arquivo de áudio e um vídeo em uma página HTML:

<img src="imagem.jpg" alt="Descrição da Imagem">

<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
</audio>

<video controls width="400" height="300">
    <source src="video.mp4" type="video/mp4">
</video>

Atributos Comuns dos Elementos de Mídia

  • src: Especifica o URL do recurso de mídia a ser exibido ou reproduzido.
  • alt: Especifica um texto alternativo para ser exibido caso a mídia não possa ser carregada.
  • controls: Adiciona controles de reprodução para o usuário interagir com a mídia.
  • width e height: Especificam as dimensões da mídia, aplicáveis apenas a <video> e <img>.

Formatos de Mídia Suportados

Os formatos de mídia suportados podem variar entre navegadores, mas os formatos comuns incluem MP3 para áudio e MP4 para vídeo.

HTML Drag and Drop

O recurso de arrastar e soltar em HTML permite que elementos da página sejam arrastados e soltos em diferentes áreas da mesma página ou em aplicativos externos.

Como Usar o Recurso de Arrastar e Soltar

Para habilitar o arrastar e soltar em elementos HTML, você precisa usar eventos JavaScript para lidar com as ações de arrastar e soltar. Por exemplo:

// Adiciona um evento de arrastar ao elemento
elemento.addEventListener('dragstart', function(event) {
    // Define o tipo de dados que está sendo arrastado
    event.dataTransfer.setData('text/plain', 'Texto a ser arrastado');
});

// Adiciona um evento de soltar ao elemento de destino
destino.addEventListener('drop', function(event) {
    // Impede o comportamento padrão de soltar
    event.preventDefault();
    
    // Obtém os dados que estão sendo arrastados
    var data = event.dataTransfer.getData('text/plain');
    
    // Faz algo com os dados soltos
    destino.textContent = 'Texto solto: ' + data;
});

Recursos do Arrastar e Soltar

  • Os eventos de arrastar e soltar incluem dragstart, drag, dragenter, dragover, dragleave e drop.
  • É possível definir os tipos de dados que estão sendo arrastados e acessá-los durante o processo de soltar.
  • Os elementos podem ser arrastados não apenas dentro da mesma página, mas também entre diferentes aplicativos, se o navegador oferecer suporte a isso.
